Deliverable: Teams workflow fully replaced with new notification service; end-to-end Airflow task monitoring alert pipeline functional again; alert rules and recipients migrated completely.

1. Context and Background
The original Teams workflow service has been shut down and no longer available, resulting in complete failure of the existing task monitoring alert system. Operation teams cannot receive real-time abnormal notifications for failed Airflow data tasks, which delays troubleshooting and increases the risk of undiscovered data pipeline breakdowns. We need to implement an alternative notification service to replace the retired Teams workflow as soon as possible to recover alert monitoring.

2. Detail Requirement
Evaluate available internal alternative notification services that support task failure alert delivery.
Migrate all existing Airflow task monitoring alert rules, trigger conditions and recipient groups from the old Teams workflow to the new service.
Reconfigure the pipeline to push task runtime errors, schedule failures and archive check exceptions to the new notification channel.
Conduct full end-to-end test: simulate failed Airflow tasks and verify alert messages can be delivered to target operation staff normally.
Document the new alert service configuration, migration steps and operation maintenance guide for future reference.

3. Actual Expectation
The retired Teams workflow is fully replaced by a stable alternative notification service.
Task monitoring alert service recovers completely; abnormal data task failures can trigger real-time notifications to operation teams without delay.
All original alert logic and receiving groups are retained and work correctly on the new service.
